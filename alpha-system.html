Design Evolution

General Changes
Removed "X" button to go back

Optimized the common use-case of going back by getting rid of the "X" symbol and replacing it with "< Back" in the top left.

Changed background image

The background image was changed to an image of a crowded indoor space which is more relevant to the applications use case.

Home Screen
Improved link to Organizer/Beacon Pairing Screen

Addressed size and clickability of link that takes you to Organizer/Beacon Pairing screen by increasing size, moving it away from "Locate People" Button, and making the "let me in" text colored as if it was a clickable hyperlink. Removed "Admin" from terminology as it confused users and replaced it with "organizer" to indicate that you are an organizer of the event.

Organizer/Beacon Pairing
Changed beacon selection interaction

Users suffered major usability issues with the check circles for selecting beacons to connect. Users wanted both the name and circle to be clickable for selection. Now the screen displays a list of beacons and their corresponding beacon IDs. Tapping on a beacon brings up a popup window with several options. This design decision was inspired by Snapchatâ€™s friend list interaction where all controls are given in a popup.

The popup seemed like the best design decision because there were so many different actions we want the user to be able to perform (edit name, connect/disconnect, view ID, view connection status). The pop up displays the name of the beacon, connection status, and ID as well as options to edit the name of the beacon, and connect the beacon. The popup is closed by clicking cancel. Tapping "Edit Name" lets you type directly into the name field. The "Save" button will also appear below. There was no more need for a "Select Beacon" button and the text "Press and hold name to rename". Pressing connect makes the beacon available to be tracked by other users. When a beacon is connected, it will be displayed as active in the list.

Tracking Screen
Removed 0 m symbol

Users did not understand what the "0 m" text and symbol meant. It did not convey any useful information so it was removed from the screen.

Objectives

Our proposal presented four objectives:

  1. the application needed to be simple and devoid of any extraneous features
  2. the system did not require prior interaction between the administrator/organizer, the locator (person looking for person of interest), and the person of interest
  3. the system did not require the person of interest to have a smartphone or any equipment prior to being made available on the application
  4. the system did not use GPS, since it needed to be effective indoors

This alpha prototype meets all four objectives. The application is extremely simple, with only 5-6 steps required to have the system ready for use. An administrator can register people of interest on the application and give them beacons, and they will instantly show up on the "locator" interface on the phones of people who have never met them. Since we use beacons as the target, the person of interest does not require a smartphone or any equipment other than the beacon assigned to them. The system uses Bluetooth signals to estimate the distance of a beacon instead of GPS, which is not effective indoors and would require additional equipment.
